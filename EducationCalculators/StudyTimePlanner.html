Study Time Planner Documentation

Introduction

The Study Time Planner is a web application designed to help students create an organized study schedule for the week. This planner allows users to input their study sessions, visualize their study time distribution, and generate a detailed timetable in PDF format.

Features

  • Weekday Selection: Users can select the day of the week for each study session.
  • Time Input: Start and end times can be set using a 12-hour format.
  • Study Session Management: Add and visualize study sessions in a table format.
  • PDF Generation: Generate a detailed timetable in PDF format with motivational quotes and total study hours.
  • Data Visualization: View a pie chart of study time distribution across different subjects.

Technology Stack

  • HTML
  • Tailwind CSS
  • JavaScript
  • Chart.js
  • pdfmake
  • Flatpickr

Installation

To set up the Study Time Planner, follow these steps:

  1. Download or clone the repository.
  2. Open the project folder and ensure you have internet access for CDN links.
  3. Open the `index.html` file in a web browser.

Usage

To use the Study Time Planner, follow these steps:

  1. Select the day of the week for your study session.
  2. Enter the subject name.
  3. Set the start and end times using the time picker.
  4. Click "Add Study Session" to add the session to your timetable.
  5. Click "Make Timetable" to visualize your study sessions in a pie chart.
  6. Click "Download as PDF" to generate a PDF of your timetable.

Code Structure

The project structure is as follows:


index.html    - Main HTML file
app.js        - JavaScript file for functionality
styles.css    - Custom CSS file (optional)

Customization

You can customize the Study Time Planner by modifying the following files:

  • HTML: Modify the structure and content in `index.html`.
  • CSS: Update styles in `styles.css` or add custom Tailwind CSS classes.
  • JavaScript: Enhance functionality in `app.js`.

Troubleshooting

If you encounter issues, consider the following solutions:

  • Ensure all required libraries are included via CDN links.
  • Check the browser console for error messages and debug accordingly.
  • Verify that the time inputs are correctly formatted and validate time logic.
